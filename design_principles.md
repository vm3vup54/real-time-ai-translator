# 介面與體驗設計原則：即時 AI 翻譯系統

這份文件記錄了讓即時語音翻譯應用程式在 UI/UX 上達到「實用且極具吸引力」，同時**確保極致效能（毫秒級延遲）**的核心設計準則。我們的目標是打造「效能視覺設計 (Performance-First Aesthetics)」。

---

## 壹、核心功能與效能融合的優先順序

即時翻譯工具在使用情境中通常處於「高壓力與注意力隨時被切換」的狀態。在兼顧效能的前提下，介面最重要的三大核心為：

### 1. 狀態可視性與系統回饋 (Visibility of System Status)
使用者對 AI 最大的恐懼是「它到底有沒有在聽？」。必須給予立即回饋，但不能耗費 CPU 資源去換取。
*   **高能效的收音確信感**：
    *   **應避免**：依賴 React State (`useState`) 驅動的高頻跳動音軌波形，這每秒數十次的重繪會嚴重搶佔處理語音的 CPU 資源。
    *   **最佳實踐**：將音量條與主介面解耦。改用 HTML5 `<canvas>` 搭配 `requestAnimationFrame`，或是使用 `useRef` 直接操作 DOM 的 `style.transform` 來繪製波形。另一種極簡解法是：偵測到聲音時，讓麥克風圖示亮起低效能消耗的綠燈。
*   **階段化無縫狀態提示**：
    *   **聆聽/轉譯中**：使用低效能消耗的微光效應（例如：邊框緩慢的透明度漸變呼吸燈），取代複雜的 SVG 迴旋動畫。
    *   **延遲遮掩 (Latency Masking)**：利用不干擾 Main Thread 的動畫（純 CSS `opacity` 過場）來填補 API 等待的焦慮空白。

### 2. 重點資訊的高對比與「字體即設計」 (Readability & Focus)
在效能優先下捨棄華麗特效，排版（Typography）就成為撐起質感的唯一支柱。創造極端對比來引導視線。
*   **極大化層級對比**：
    *   **結果（目標翻譯）**：給予極度明確的打擊感。字體要巨大、粗體（如 `font-semibold text-4xl`），甚至使用純白或帶微光澤的顏色，這是畫面的絕對主角。
    *   **原文（來源文字）**：作為微小的輔助核對資訊。像浮水印一樣，使用較小字體與低對比色彩（如淺灰色 `text-zinc-500`），不喧賓奪主。
*   **高能效閱讀體驗**：拋棄大面積的毛玻璃（`backdrop-filter`，極度消耗 GPU）。改用高對比的「深底（如 `bg-zinc-950`） + 局部光暈 + 邊緣微亮內陰影」，達成科技質感但零效能負擔。

### 3. 降低認知負擔與 DOM 控制 (Cognitive Load Reduction)
*   **自動化取代點擊**：不讓使用者分心操作。高度依賴 VAD（語音活動偵測）來自動切分句子。整個介面應該只有一個「開始工作」的總開關。
*   **嚴格控制 DOM 數量與虛擬化 (Pruning)**：
    *   對話時間拉長會累積大量 HTML 節點，導致頁面越來越卡。
    *   **最佳實踐**：在「即時字幕」模式，嚴格限制畫面上永遠只保留最近 5~10 句話。更老的對話無情從 DOM 中移除（但這不影響匯出功能），維持最淺的 DOM 樹以達到物理極限的順暢。
*   **緊急停止機制 (Force Stop)**：在顯眼處提供一鍵「閉嘴 / 中斷」的防呆設計，處理雜音失控或 TTS 長篇大論的情況。

## 貳、高級視覺美學實作法則 (Performance-First Aesthetics)

要在不動用重度 CSS 濾鏡的前提下，讓人有「這工具好酷、好專業」的感覺，請遵守以下黃金守則：

*   **動畫只准動「絕對安全」屬性 (Hardware Acceleration ONLY)**：
    *   **鐵律**：所有的過場動畫，只能操作 `transform` (`translate`, `scale`) 與 `opacity`。這些屬性由 GPU 處理，順滑且不佔 CPU。
    *   **嚴禁**：用動畫改變歷史卡片的 `height`、`width` 或 `padding`，避免引發致命的瀏覽器重排 (Reflow)。舊卡片被往上推時，直進直出或順滑位移即可。
    *   **實例**：翻譯字句定案時，讓字體做個輕微的 `scale(0.98)` 到 `scale(1.0)` 的放大回彈動畫，能營造極佳的高級確認感。
*   **用「光影」取代「模糊」**：
    *   放棄背景模糊，改以深色純背景搭配邊緣 1px 的細微高光（`inset` 陰影），或在重點狀態的邊框處加入流動的漸層色彩，營造深邃且俐落的科技氛圍。
*   **減法設計**：
    *   讓出 90% 的呼吸空間。把設定選項、選單、按鈕等做成無框風格，甚至未選中時降低不透明度（變暗），僅在滑鼠經過時浮現。讓畫面全留給核心文字。
